# タイミングチャートアプリ MVP

## 今回の変更
- 機器一覧を 1 つの表に統合
- 表の列を以下に変更
  - 大項目ID
  - 大項目
  - 中項目ID
  - 中項目
  - 小項目ID
  - 小項目
  - 動作種別
  - 動作番号
  - 動作
- 階層IDをレベルごと・親ごとに管理
  - 中項目IDは大項目ごとに連番
  - 小項目IDは中項目ごとに連番
  - 親が違えば同じ番号を使える
- 名称のインデントで階層を表現
- 動作定義一覧を別表ではなく機器一覧へ統合
- グラフの依存矢印を依存関係の始点・終点を直接結ぶ形に変更
- Undo / Redo は継続

## セットアップ
```bash
pip install -r requirements.txt
python timing_chart_app.py
```

## 注意
- 内部では一意識別用に UID を持っています。
- 画面で扱う ID は、階層ごとの表示用 ID です。
- 旧JSON読込にもある程度対応しています。


## 今回の見た目調整
- 機器一覧は列を簡素化し、項目を1列に集約
- 各名称の先頭にIDを付与
- 大項目 / 中項目 / 小項目は改行＋インデントで階層表現

- 機器一覧をツリー表示に変更し、階層ごとに折りたたみ可能に調整
- 動作は小項目の子ノードとして表示
